클래스의 생성자를 정의할 때, 보통 사용할 변수와 함수의 프로토타입을 헤더파일에 적는다.

```c
Physics::Physics(float initial_position)

{
	position = initial_position;
	velocity = 0.0f;
	g = 9.81f;
	beam_length = 0.5f;
} 
```

뾰롱

근데 이건 헤더파일이 아닌 소스파일에서 클래스 함수를 짜기 전 선언한 생성자이다.

### 왜이렇게 나눠서 선언했을까?

1. 설계도와 부품을 분리하는 것처럼, 헤더파일은 목록만 보여주는 역할이고 소스파일은 그 기능에 필요한 부품을 적어주는 역할이다
2. 그리고 모든 코드를 헤더에 때려박으면 그 헤더를 포함하는 파일이 수정될 때마다 다시 필요없는 것도 다시 컴파일해야 해서 시간이 걸린다.
3. 이렇게 하면 코드의 가독성도 좋아진다.

 

### 그리고 자료형을 적지 않은 이유

생성자는 원래 반환타입이 없기 때문이다.

생성자는 객체를 생성하는 것 자체가 목적이므로, 아무것도 반환하지 않는다. (걍 구조체다.)
