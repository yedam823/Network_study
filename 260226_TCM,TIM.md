## TCM 이란?

Tightly Coupled Memory

CPU와 직접 연결되어 있는 영역

그래서 다른 메모리 관리 단계를 거치지 않아 항상 접근시간이 일정하다.

cortex-M, ARM에서 많이 사용한다고 한다.

- ITCM : 빠른 명령어 실행
- DTCM : 빠른 데이터 접근

## TIM 이란?

TCM과 유사하게 접근 속도를 높이는 역할인데,

일반적으로 캐시를 통해 접근한다.

ITIM, DTIM 존재

### 캐시 접근으로 인한 차이?

캐시 특성을 보면 접근 속도는 높여주지만 미스가 나면 메모리 대역폭 경합으로 시간 예측이 어렵다.

그렇기에 일정 접근 시간이 보장된 실시간 응답 시스템에서 TCM를 사용한다. (항공기 제어, 의료기기)

반면, 안전성은 떨어지지만 성능 최적화에 목적을 둔 TIM은 비디오 디코딩/인코딩, 네트워크 패킷처리, 머신러닝 등에 사용한다.

### 캐시와의 차이점?

공부하다가 둘다 장점이 빠른 접근이고 SRAM인데 뭔 차이가 있냐는 생각이 들었다.

알고보니 TCM은 사용자가 직접 메모리 주소를 할당하고 관리하지만, 캐시는 하드웨어가 자동으로 할당하는 것이기에 예측 가능성이 TCM이 압도적으로 높다는 차이가 있었다.

또한 메모리는 CPU → 캐시 → bus → 메인메모리 로 접근하는데, TCM은 Bus를 사용하지 않기 때문에 병목현상도 일어나지 않는다.

### 단점

이렇게 좋은데 단점이 없을 수가 없지

사용할 수 있는 메모리량이 매우 한정적이다.

보통 아주 중요한 ISR, 실시간 스택, 크리티컬 데이터를 저장한다.
