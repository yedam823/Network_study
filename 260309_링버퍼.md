## 링버퍼?

원형큐의 다른말이다

읽은 위치와 쓰는 위치를 따로 관리하는 원형 버퍼이다.

원형큐의 로직은 알기때문에 자세한 설명은 넘어가겠다.

### 왜쓰냐?

이건 중요하다.

UART data를 버퍼에 넣고 그걸 그대로 전송해버리면 data출력이 덜 출력된다.

왜? UART 데이터를 읽어서 보낼때는 인터럽트를 사용한다. 근데 중간에 또 인터럽트가 걸려버리면 CPU가 처리하기 전에 새 데이터가 덮어쓰기 때문에 문제가 생긴다.

예 ) A가 와서 A를 처리중 B와 C가 왔다. 그러면 덮어씌워져서 A값이 사라지고 B C 값만 쓰게 되는 것이다.

이래서 링버퍼를 쓴다

### 장점

- 포인터 2개를 사용한다는 것이다.

예) write, read 포인터 두개를 사용한다고 가정했을 때, write 포인터를 써서 UART을 받을 수신버퍼에 data를 write 한다. 그리고 data를 read 포인터로 읽어서 출력한다.

→ 그러면 인터럽트가 걸렸을 때, write 포인터에 의해 버퍼에 data만 기록하게 되고, UART data를 다 보낸 다음에 다시 read 로 읽어서 보내면 된다. 

- 데이터 처리를 인터럽트에서 하지 않는다?…

→ 한마디로 원형큐 형식으로 데이터 저장만 하고 처리는 메인에서 하는 것이다. (치고 빠지기)
