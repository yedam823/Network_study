## modbus?

매우 널리 쓰이는 프로토콜

직렬 통신으로, 공장자동화에 사용되는 PLC 통신을 위해 만들어졌다.

- PLC?
    
    PLC 통신이란, 공장에서 기계들이 서로 데이터를 주고받을 수 있게 하는 통신이다.
    

1대의 마스터에서 다수의 슬레이브를 제어하기 위한 약속

### 종류

- Modbus RTU (Remote Terminal Unit) : 바이너리 데이터형 전송방식
- Modbus ASCII : 아스키문자를 사용하는 텍스트형 전송방식
- Modbus TCP/IP : TCP/IP 계층에서 사용되는 RTU 비슷 방식

### RTU 방식

바이너리 통신을 하는 방식

공백으로 프레임을 구분

16진수 문자를 직접 사용하므로 통신 시간 단축이 된다.

### RTU 구성요소

- bitrate(bps, bit per second) : 초당 전달되는 비트의 수, 통신 하는 기기가 주기가 맞아야함
- 데이터 길이 : 주로 8비트 사용
- 패리티 비트 (N : 안쓴다는 뜻, E : 짝수, O : 홀수)
- 정지 비트 : 주로 1비트 사용
- 흐름 제어 : 버퍼가 꽉찼을 때 멈춰달라고 하는 기능 (보통 사용X)

### 시리얼 통신

RS-232 : 1:1 통신, 노이즈에 약함

RS-422 : 전이중 통신

RS-485 방식 : 가장 많이 사용, 여러개의 단말기들을 하나에 연결 가능

### UART 와 다른점?

UART는 단순히 데이터를 전송하는 방식인거고,

안의 내용이 무엇인지 이해하지 못한채 그냥 보내는 역할이다.

하지만 modbus는 데이터가 무슨 뜻인지 Hex 로 표기하는 하나의 약속이다.

예) 01 03 00 00 02

1번 장치, 읽기요청, 주소, 1개읽기 라는 메세지를 만들고,

이를 UART로 RS-485, RS-232 등의 전기 신호 규격 선에 맞춰 전송한다.

### modbus 통신의 특징

- Master는 무조건 한개, Slave는 최대 247개 연결 가능
- 최대 통신 가능 거리 : 1km
