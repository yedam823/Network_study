## 스케줄링이란?

프로세스가 생성되어 실행될 때 필요한 시스템의 여러 자원을 해당 프로세스에게 할당하는 작업

CPU는 한번에 하나의 작업만 처리 가능한데, 여러개의 작업이 존재할 경우 해당 작업들이 번갈아 가면서 수행됨 → 스케줄링

프로세스 실행되는 순서를 정해주는 작업이기도 함

### 스케줄링 과정

1. 문맥교환
    1. 현재 CPU가 할당된 프로세스의 정보를 저장 → 다음 프로세스 선택 → CPU 할당 후 실행
    2. 여기서 기존 프로세스 정보를 PCB (Process Control Block) 에 저장함

### 언제 스케줄링이 일어나는가?

1. 새로운 프로세스가 생성되었을 때
2. 프로세스 종료 시
3. 프로세스가 I/O, 세마포어 등으로 대기
4. I/O 인터럽트가 발생에 현재 프로세스가 중단 되었을 때

### 스케줄링 방식

1. 선점형 스케줄링 (Preemptive scheduling)
    - 어떤 프로세스가 CPU를 점유하고 있어도 다른 프로세스가 실행중인 프로세스를 중지하고 CPU를 강제로 점유할 수 있음 → 새채기 가능!
2. 비 선점형 스케줄링 (Non-preemptive scheduling)
    - 실행중인 프로세스는 종료 또는 I/O 요청으로 block 상태로 갈 때 까지 중단되지 않음
    - 어떤 프로세스가 CPU를 점유하고 있다면 그 프로세스가 종요되거나 입출력 요구가 발생하여 자발적으로 중지될 때까지 계속 실행되도록 보장됨
    - 일괄처리 시스템에 적합함

### 스케줄링 목표

- Fairness : 각각의 프로세스에 공평하게 CPU 할당
- balance : 시스템의 모든 부분을 사용 상태로 유지

### 우선순위 스케줄링 (Priority Scheduling)

사용자에 따라 다른 우선순위 할당

- 기아현상을 방지해야함
- 우선순위 정적인 할당 : 교수 100 선생 80 학생 60 …
- 우선순위 동적인 할당 : I/O 요청 등으로 time quantum을 다 사용하지 않은 경우, 더 높은 우선순위로 조정함

예) time quantum이 50인데 25를 사용했다면 우선순위는 50/25

![image.png](attachment:edf8852b-4603-4999-8142-db80bf407594:image.png)

우선순위가 높은 프로세스부터 실행 수가 높을수록 우선순위가 높은 것

기아현상이 일어나지 않도록 우선순위 조정이 필요함

> MLFQ (Multi-Level Feddback Queue)
> 

![image.png](attachment:2f9a5c1e-49a1-44de-b203-cf15e3760a3b:image.png)

1st level의 프로세스를 실행시켜주며 주어진 Time quantum을 다 사용하면 2st level을 실행시켜줌

점점 우선순위가 낮은 프로세스를 실행시키는데 우선순위가 낮을수록 time quantum이 많아짐
