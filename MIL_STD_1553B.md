## MIL-STD-1553B

군사용 항공전자 시스템을 위해 설계되었지만, 산업용, 우주용 장비의 데이터 통신으로 폭넓게 사용되고 있는 통신 프로토콜

서로 다른 전자장비가 장착될 경우 전기적 인터페이스로 인한 혼란을 방지하기 위해 메세지 전송 방식의 통신을 한다.

신뢰성이 매우매우 중요한 곳에서 사용한다.

### 구조

통신망 구조 : 버스토폴로지 방식

통신 방식 : 반이중 전송방식 (한쪽으로만 보내기)

속도 : 1MHz (1Mbps)

(속도가 좀 느리다고 생각했는데, 군사용에서는 속도보다 생존이랑 정확성이 중요하기 때문에 이렇다네요)

### 특징

- 데이터 타입 : 디지털 형태이며, 시분할 방식을 사용하여 데이터 스케줄링, command, reponse 로 동작한다.
- 터미널 넘버를 부여하여 최대 31개의 터미널 운영이 가능하다.
- 1553B 버스는 BC(Bus Controller), RT(Remote Terminal), MT(Monitoring; BM) 로 구성된다.
    - BC : 모든 제어와 스케줄링 담당, 모든 데이터는 여기 command에서 RT로 보냄으로써 수행된다.
    - RT : BC 로부터 받은 명령을 수행하고 상태를 BC에 보고한다.(ACK)
    - MT : 송수신 데이터들을 확인한다.

### 구성

하나의 메인버스 + 여러 터미널

- Main bus : 실제 데이터가 흐르는 선
    - Redundancy(이중화)로, 버스 두개 사용
- Stub : 메인 버스에서 각 장치로 분기되는 선
    - 20ft 이내로 제한하는 특징이 있다.
- Terminator : 신호 반사를 막기 위해 버스 끝에 설치
    - 신호반사? : 전기 신호가 전송선을 따라 이동하다가 되돌아오는 현상 (임피던스 불일치로 인해) → 이게 생기면 신호가 왜곡된다.
- Coupler : 메인버스와 터미널을 연결하는 장치

### 데이터 포맷

Word 단위로 통신한다.

- 크기 : 20bit
- 구성 : Sync [3bit] + Data[16bit] + Parity[1bit]
- 데이터 인코딩 : Manchester || BI-phase
    - Manchester? 중간에 전압 신호를 바꿔서 클럭 동기화를 쉽게 하는 것
    - 클럭과 데이터를 동시에 전송하고(클럭선 필요X), 노이즈에 강하다는 특징이 있다. (참고로 문서에 저렇게 써져있고 Manchester가 BI-phase의 한종류임)

Word 구조는 command word, data word, status word가 있다.

1. command word : BC가 RT에게 명령 내릴때 사용
2. data word : 데이터 전달할 때 사용
3. status word : RT에서 BC로 상태 보낼 때 사용

→ 이걸 묶어서(Message) 보낸다.

### Manchester는 왜 2배의 대역폭이 필요한가?

일반 인코딩과는 다르게, 무조건 비트마다 변환해야한다는 특징이 있어 변환속도가 추가된다.

원래 1MHz 였지만, 이걸 사용하기 위해서는 2MHz를 사용해야한다는 것이다.

그래서 대역폭이 2배로 는다. (데이터 속도는 1Mbps로 표기되어 있지만 실제 신호 주파수는 2MHz 이다.)
