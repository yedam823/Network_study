## 동시성

하나의 processor가 여러 task를 동시에 처리하는 것처럼 보이게 하는 것

스레드를 통해 2개이상의 작업을 아주 짧은 시간으로 나누고 번갈아 처리함 → 여러 작업이 동시에 수행되는 것처럼 보임

![image.png](attachment:f7f73d79-8055-4eec-8ce1-4f6243103756:image.png)

- 경합 상태 (Race Condition)
- 데드락()

### 멀티태스킹

프로세스가 한번 CPU를 사용할 때 아주 짧은 시간만 실행되도록 하여 각 프로세스의 작업을 아주 작은 단위로 번갈아가며 처리하며 작업 응답 시간을 최소화 시키는 것

이때 잘게 나뉘어진 프로세스끼리 스위칭 되는 것 → 문맥교환

### 동시성과 멀티태스킹은 같은 말일까?

다른 용어.

- 멀티태스킹 : CPU 관점에서 각 프로세스를 컨텍스트 스위칭 하며 처리하는 방식
- 동시성 : 프로세스 관점에서 각 task를 스레드를 통해 동시에 실행하는 것처럼 보이는 방식
- 멀티 태스킹 > 동시성

## 병렬성

컴퓨터에 두개 이상의 CPU 또는 CPU 코어가 있고, 동시에 두개 이상의 작업을 진행하는 경우

실제 CPU에서는 멀티코어를 사용하기에 여러 task를 동시에 처리 가능함

![image.png](attachment:a682189f-2237-4c36-9b26-1313e3b85aef:image.png)

### 동시성과 병렬성의 차이

| 구분 | 동시성 | 병렬성 |
| --- | --- | --- |
| 개념 | 동시에 처리하는 것처럼 보이게 하는 것 | 여러 작업을 실제로 동시에 처리하는 것 |
| 사용 코어 수 | 싱글 코어 | 멀티 코어 |
| 동작 방식 | 싱글 코어에서 멀티 쓰레드 | 멀티 코어에서 멀티 쓰레드 |
| 개념적 차이 | 논리적인 개념 | 물리적인 개념 |

## 동기(synchronous)

순차적으로 진행된다

하나가 시작되면 해당 작업이 끝날때까지 기다려야 함

반환될 때까지 대기하는 동안 실행흐름이 차단됨

- 일반적으로 간단, 직관적
- 하지만 여러작업이 동시에 실행되어야하는 경우 각 작업의 완료를 기다리는 동안의 시간이 소요되어 전체 프로세스의 성능이 저하됨

## 비동기(asynchronous)

순차적으로 진행되지 않는다

한 작업이 시작되면 해당 작업이 완료될 때까지 기다리지 않고, 다른 작업을 실행함

이러한 특징 떄문에 비동기 방식은 주로 I/O 작업이나 네트워크 요청과 같이 시간이 오래걸리는 작업에 사용됨

- 프로그램은 작업이 완료되길 기다리는 동안 다른 작업을 처리항 수 있으므로 전체적 성능이 향상됨
- 콜백(callback), 프로미스(promise), async/await 등의 메커니즘을 통해 구현될 수 있음
