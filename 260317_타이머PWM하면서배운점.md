## Period(ARR), Prescaler(PSC) 원리

- Prescaler(분주기) : 클럭의 속도를 늦추기 위해 사용
    - 클럭을 얼마나 나눌것인가
    
    $$
    f_{out} = (f_{clk}) /(PSC + 1)*(ARR+1)
    $$
    
- ARR(Auto Reload Register/Period) : 몇번까지 카운트 할것인지
    - 만약 저기서 f_{clk} 가 84MHz 일때 딱 1MHz 를 만들고 싶다면 84M이 되어야 하는 것임
        - 만약 PSC가 8400이라 하면 ARR은 10000

## CCR

Capture / Compare Register

입력, 출력을 수행하는 레지스터

- PWM에서 많이 사용된다.
    - Compare : 타이머의 현재 CNT 값과 CCR 설정한 값을 계속 비교하면서, CRR값보다 작을 때는 HIGH, CCR 값에 도달하면 LOW로 바뀜
        - 이걸로 CCR 값을 조절하면서 듀티비 제어가능
    - Capture : 신호가 들어오면 그때 타이머 CNT 값을 CCR 레지스터에 복사해 저장한다.
        - 신호가 들어온 시점의 값을 알 수 있고 캡처 값의 차이를 계산해 펄스 폭을 알수있다.
        
        → 이벤트가 발생할 시점을 결정한다. 신호를 언제 꺾을지 정하는거임
        
        예시) 500이면 50%의 LED 밝기, 900이면 90%의 LED밝기, 100이면 10%의 LED 밝기
        
        그래서 유동적으로 PWM을 제어가능하다.
        
        ```c
        __HAL_TIM_SET_COMPARE(&htim4, TIM_HANNEL_1, ccr);
        ccr += 1000;
        if (crr > TIM4->ARR) crr = 0;
        HAL_Delay(50);
        // STM32 코드
        ```
