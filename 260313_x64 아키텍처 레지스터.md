## x64 아키텍처 레지스터

x64 레지스터들은 8개의 범용 레지스터와 8개의 추가 레지스터로 구성된다.

### 범용 레지스터

RAX /RBX / RCX / RDX

함수 계산을 할 때 주로 사용되는 레지스터

- RAX : 함수 리턴 값을 저장한다.
- RBX : RSI, RDI와 연계하여 기준 주소의 오프셋을 나타내는 값을 저장한다.
    - RSI와 RDI : 복사할 데이터, 붙여넣을 데이터의 주소를 저장하는 레지스터
- RCX : for, while문에서 자주 사용되는 반복 횟수가 저장된다.
- RDX : RAX와 같이 범용으로 쓰인다.

### 스택 포인터 레지스터

RBP / RSP

- RBP : 스택 메모리의 시작 주소를 저장한다.
- RSP :  증가하는 스택 메모리의 끝주소를 저장한다.

### 인덱스 레지스터

RSI, RDI

복사나 비교대상이 자주 되는 주소의 시작, 목적지를 저장한다.

### 인스트럭션 포인터 레지스터

RIP

다음 명령어의 위치를 저장한다.

### E-플래그 레지스터

EFL

CPU의 상태를 나타내는 레지스터

주로 중간 계산 결과에 대한 CPU 상태를 반영한다.

### 추가 레지스터

R8 ~ R15

용도가 다양한 레지스터다.

어셈블리 분석으로 통해 용도를 알면, Memory Latency도 줄일 수 있고, context switching시의 오버헤드도 계산할 수 있다.

### 내가 어떻게 써먹을 수 있을까?

- RAX, RBX : 반환값을 확인하거나 곱셈/나눗셈같은 계산 결과를 검증할 수 있다.
- RIP : 시스템 crash가 발생했을 때 문제 근원지를 찾을 수 있다.
- RSP, RBP : 스택 오염을 감지하거나 콜스택 복원이 가능하다.
- R8~R15 : 최적화 효율을 확인하거나 레지스터를 변수로 굴리기
