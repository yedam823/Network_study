## 후킹이란?

API의 호출 흐름을 중간에 가로채는 것

원래 진행대로 안가고 후킹 함수가 추가로 실행되어 이쪽에서 처리후에 API를 호출한다.

## LD_PRELOAD?

프로그램을 실행할 때 내가 지정한 라이브러리를 가장 먼저 메모리에 불러오라고 명령하는 기능

### 작동원리

원래는 프로그램이 실행될 때 기본 라이브러리들을 메모리에 Load 해서 그 안에 있는 함수를 가져다 쓴다.

하지만 LD_PRELOAD 환경 변수를 설정하면 지정된 사용자의 커스텀 라이브러리를 1순위로 load한다.

그다음에 기본 라이브러리들을 load 한다.

### 이걸로 뭘 할 수 있나?

- 코드를 수정하거나 다시 컴파일 하지 않고도 프로그램의 동작을 내맘대로 후킹할 수 있다.
    - 예시) malloc() 이나 free() 같은 메모리 함수들을 가로채서 메모리가 얼마나 할당되고 어디서 누수가 발생하는지 추적하는 도구를 만들 수 있다.

### 해킹의 위험성?

너무 위험하지 않나? 라는 생각을 했다. 악의적으로 탈취할 수 도 있을거 같지만

리눅스에서는 루트 권한으로 실행되는 프로그램(SetUID, SerGID)이나 권한이 부여된 파일 (sudo, passwd)을 실행할 때는 LD_PRELOAD 변수를 완전 무시하게 되어있다.

그래서 걍 일반 유저 권한에서만 조작이 가능하기 때문에 위험하진 않는다고 한다.(?)

### strcmp 치환

C나 C++로 만들어진 프로그램은 사용자의 비밀번호가 키를 확인할 때 strcmp를 자주 쓰는데, 여기서 해킹을 당할 수 있다. (무조건적으로 맞으면 0을 반환하기 때문에)

- 대표적으로는 후킹을 이용해 strcmp 함수를 어떤걸 치든지 0을 반환하게 할수있다.

```c
int strcmp(const char *s1, const char *s2) {
    // 실제 비교는 하지 않고 무조건 0을 던져버림
    return 0; 
}
```

- 또한 진짜 비밀번호가 들어온다는 점을 이용해 몰래 로그를 찍는다.

```c
int strcmp(const char *s1, const char *s2) {
    // 비교하라고 넘어온 두 개의 문자열을 화면에 몰래 출력해 버림!
    printf("[해킹 완료] 문자열 1: %s\n", s1);
    printf("[해킹 완료] 문자열 2: %s\n", s2);
    
    return 0; 
}
```
