## TCP란?

IP 프로토콜 위에서 연결형 서비스를 지원하는 전송계층 프로토콜

인터넷 환경에서 기본으로 사용하며 전이중 방식의 가상회선이다.

3 핸드셰이크로 신뢰성 굳!

패킷의 단위는 세그먼트이다.

### 헤더 구조

최소크기는 20바이트, 최대크기는 40바이트이다.

| 보내는 포트 | 2B |
| --- | --- |
| 받는 포트 | 2B |
| 시퀀스 넘버 | 4B |
| 확인 응답 넘버 | 4B |
| data offset(헤더 길이) | 4bit |
| reserved | 3bit |
| 여러 플래그 (ECE, CWR, URG, ACK, PSH, RST, SYN, FIN) | 9bit (위까지 합치면 2B, 16bit) |
| 윈도우 사이즈 | 2B |
| 체크섬 | 2B |
| Urgent Pointer (긴급 데이터 위치) | 2B |
| 옵션/사이즈 맞추기 위한 패딩 | 가변 |

순서관리, 재전송, 흐름제어, 혼잡제어… 이런거 알아서 다 해줌 (시퀀스 넘버가 순서 보장, 확인 응답 넘버가 재전송 판단기준)

### 플래그 정리

- URG : Urgent pointer 필드가 유효한지를 나타냄
- ACK : 확인 응답 넘버 필드가 유효한지를 나타냄
- PSH : 현재 세그먼트에 포함된 데이터를 상위 계층에서 즉시 전달하도록 지시할 때 사용
- RST : 연결의 리셋이나 유효하지 않은 세그먼트에 대한 응답으로 사용
- SYN : 연결 설정 요구를 의미하는 플래그 (가상 회선 연결 설정할 때 사용)
- FIN : 한쪽 프로세스에서 더이상 보낼 데이터가 없어 연결 종료하고 싶다는 의사 표시를 상대방에게 알림
