## Private?

이걸 왜쓸까?

→ 설계자의 의도를 전달할 수 있기 때문이다.

예시) count라는 변수가 있을 때, 이 변수가 전역변수로만 남아있을 때는 누구든지 접근해서 맘대로 바꿀 수 있음. 그래서 의도를 모를 때가 있음

→count를 private으로 숨기고 count가 들어있는 메소드를 public으로 공개함으로써 프로그램이 의도대로 동작할 수 있게끔 만든다.

### 구현을 위해서

private는 메모리 정렬과 타입캐스팅을 이용한다.

메모리 정렬? 해당 변수의 주소 위치부터 정확히 접근하기 위해서

타입캐스팅? 컴파일러에게 해당 변수를 이정도 크기로 해석한다는 걸 알려줌

```c
struct Output{
	void (*TurnOn)(Output*);
	void (*TurnOff)(Output*);
	void (*Toggle)(Output*);
	
	
	GPIO_TypeDef* port;
	uint16_t pin;
	OutputType type;
};
```

이 코드 같은 경우, Output 변수를 만들면 함수포인터가 아닌 port, pin 이런거까지 접근할 수 있다는 단점이 있음

보통 C++에서는 이런 변수를 private로 나눠서 관리함

이럴 땐 구조체 하나를 더 추가해줌

```c
struct _Output{
	void (*TurnOn)(Output*);
	void (*TurnOff)(Output*);
	void (*Toggle)(Output*);
};

typedef struct{
	Output public;
	
	GPIO_TypeDef* port;
	uint16_t pin;
	OutputType type;
}OutputPrivate;
```

이런식으로 이루어짐, 그리고 타입캐스팅으로 Output까지만 접근할 수 있게 함

```c
static Output led = (Output*)&ledPrivate;
// 접근할 수 있는 건 함수 포인터 뿐이게 됨
```

## 싱글턴?

돌아가고 있는 프로그램에서 이 객체는 단 하나여야한다.

예시) 다크모드는 싱글턴으로 구현한다. 모든 페이지가 똑같이 다크모드가 적용될 수 있게 하나로 관리

임베쪽에서는 HMI 터치센서(스크린), 디버그용 부저 같은 것을 이렇게 관리하기도 한다. (하나만 있어도 되니)

인버터 싱글턴을 구현해보려 한다.
