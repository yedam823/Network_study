## RISC

명령어를 단순하게 만들어서 빠르게 처리하자는 설계철학

다양한 코어들이 RISC 철학을 사용한다.

## RISC-V

RISC 설계철학을 바탕으로 만들어진 오픈소스 CPU 아키텍처

요즘 정말 많이 뜨고있다

오픈소스라 공짜라는 점이 다른 아키텍처와의 차별점이다.

### 왜 V가 붙나?

버클리 대학에서 연구하고 있던 아키텍처인데 그게 5번째라 로마자 V가 붙게된것이다.

### 특징

- 모듈형 아키텍처라서 기본 세트에 필요한 기능을 조절할 수 있다.
- 메모리 접근 명령어를 가진 load-store 구조를 사용한다.
- 모든 연산 조작을 레지스터로만 할수있다.
- 시스템 특권 모드는 3가지이고, 스레드나 인터럽트를 유연하게 다룰 수 있다. (User, Supervisor, Machine)
    - M-mode : 우두머리, 인터럽트나 예외처리를 직접 처리하거나 S로 넘기고, CPU 전체를 관리한다 생각하면 됨

### 로직짤 때의 장점

- x0 이라는 레지스터가 있는데 기본적으로 0이 박혀있어서 x64처럼 RAX같은 연산을 굳이 하지 않아도 된다.
- RISC는 명령어가 단순해서 자칫 잘못하면 코드 크기가 커지는데, -V는 명령어를 16비트로 불여주는 장점이 있다. (용량 줄임, 가변적)

### 단점

- 파편화가 심하다. (근데 ARM도 파편화 문제가 있기에… 쌤쌤?)
- 딱히 성능이 엄청 좋진 않다.

### 레지스터 구성

| 레지스터 이름 | ABI 이름 | 용도 |
| --- | --- | --- |
| x0 | zero | 항상 0을 반환 |
| x1 | ra | 복귀 주소 (Return Address) |
| x2 | sp | 스택 포인터 (Stack Pointer) |
| x3 | gp | 전역 포인터 (Global Pointer) |
| x4 | tp | 스레드 포인터 (Thread Pointer) |
| x5 | t0 | 임시 레지스터 / 대체링크 레지스터 |
| x6 ~x7 | t1 ~ t2 | 임시 레지스터 |
| x8 | s0 / fp | 저장된 레지스터 / 프레임 포인터 |
| x9 | s1 | 저장된 레지스터 |
| x10 ~ x11 | a0 ~ a1 | 함수 인자 / 반환값 |
| x12 ~ x17 | a2 ~ a7 | 함수 인자 (4개) |
| x18 ~ x27 | s2 ~ s11 | 저장된 레지스터 |
| x28 ~ x31 | t3 ~ t6 | 임시 레지스터 |

ABI : 일반 응용 프로그램에서 함수 호출 시에 레지스터를 어떻게 사용할지에 관한 규약이다.

제어 레지스터, 타이머 등의 특수 레지스터를 CSR 라고 부른다. CSRRW, CSRRS 등의 명령어로 접근한다.
