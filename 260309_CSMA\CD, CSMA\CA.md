## **CSMA/CD?**

이더넷 환경에서 사용하는 통신

Carrier Sense Multiple Access/Collision Detetion

‘대충 알아서 눈치껏 통신하자’

여러 디바이스가 동시에 데이터를 송신하게 되면 충돌이 발생하고 이 충돌을 해결하기 위해서 CSMA/CD가 존재한다.

### 방식

1. 이더넷 환경에서 네트워크 상에 통신이 일어나는 지 확인한다. (캐리어 있는지 검사한다.)
    - 캐리어란?
        
        네트워크 상에 나타나는 신호
        
    - 캐리어가 감지되면 데이터를 보내지 않고 기다린다.
    - 감지되지 않으면 데이터를 보낸다.
2. 만약, 캐리어가 감지되지 않았을 때, 두 pc나 서버가 데이터를 동시에 보내면 이 경우를 Multiple Access 이라 한다.
3. 위처럼 두 PC나 서버가 데이터를 동시에 보내려다 부딪히는 경우 충돌이 발생했다고 한다.
4. 만약 충돌이 일어나게 된다면 데이터를 전송했던 두 PC나 서버들은 랜덤한 시간동안 기다린 다음 다시 데이터를 전송한다.

## CSMA/CA?

주로 무선으로 활용된다.

충돌 감지를 하는 CD와 달리 CA는 충돌 회피를 한다.

충돌 감지가 불가능 하기 때문이다

즉, LBT (Listen Before Talk) 방식 인거다

- LBT?
    - 만일 채널이 비어있으면, 자신의 패킷을 송출 전송하고
    - 사용중이면, 채널이 빌 때까지 합리적인 방식

![image.png](attachment:89863ab6-b51d-4b39-ba7d-bd460a881837:image.png)

이 사진처럼 C의 탐지범위에 A가 없다. 이렇게 A는 C의 hidden node 가 되는데, C입장에서는 A가 통신을 하는지 모르기 때문에 CD가 불가능 한 것이다.

### 방식

RTS/CTS 방식을 통해 문제점을 보완한다.

- 송신을 원하는 주체가 Request to Send를 보내면, 수신 주체가 통신이 가능할 때에만 Clear to Send를 한다.

![image.png](attachment:2ae3688f-e4b0-46b3-b418-9e3ea507c140:image.png)

- CTS 프레임을 받지 못하면 일정 회수만큼 RTS를 다시 보내고 그대로 받지 못하면 일정시간 대기 후에 다시 보낸다.
