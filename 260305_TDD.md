## TDD?

테스트 주도 개발

작은 단위의 테스트 케이스를 작성하고, 이를 통과하는 코드를 추가하는 것

![image.png](attachment:5f193901-f114-4a4b-82dd-be388c41bedc:image.png)

- Red 단계에서는 실패하는 테스트 코드를 작성함
- Green 단계에서는 테스트코드를 성공하기 위한 실제 코드를 작성함
- Blue 단계에서는 리팩토링을 함

### 중요한 것

1. 실패하는 테스트 코드를 작성할 때까지 실제 코드를 작성하지 않음
2. 실패하는 테스트 코드를 통과할 정도의 실제 코드를 작성해야함

### 장점

실제 코드에 대해 기대되는 바를 명확하게 정의함으로써 불필요한 설계를 피할 수 있고, 정확한 요구사항에 집중할 수 있다.

### 일반 개발과의 차이

<aside>

설계 → 코드개발 →테스트

</aside>

위에 것이 일반적인 개발 흐름이고

<aside>

설계 → 테스트 코드 작성 ↔ 코드개발, 리팩토링

</aside>

아래 것이 TDD 개발이다.

- 차이점 : 테스트 코드를 작성한 뒤에 실제 코드를 작성한다.
- 그래서 설계 단계에서 목적을 반드시 정의해야한다.
- 또 무엇을 테스트해야 할지 미리 정의해야한다.
