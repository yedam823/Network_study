## DHCP란?

동적 주소 할당 프로토콜

호스트의 IP주소와 각종 TCP/IP 프로토콜의 기본 설정을 클라이언트에게 자동적으로 제공해주는 프로토콜이다.

- 네트워크에 사용되는 IP주소를 DHCP서버가 중앙집중식으로 관리하는 클라이언트/서버 모델을 사용한다.
- DHCP 지원 클라이언트는 네트워크 부팅과정에서 DHCP 서버에 IP 주소를 요청하고 얻을 수 있다.
- 네트워크 안에 자동으로 네임서버 주소, IP주소, 게이트웨이 주소를 할당해준다.

### 장점

PC 수가 많거나 PC 자체 변동사항이 많으면 IP 설정이 자동으로 되기 때문에 효율적으로 사용 가능하고, IP 충돌을 막을 수 있다.

### 단점

서버에 의존을 많이 해서 서버가 다운되면 IP 할당이 안이루어진다.

### 구성요소

1. DHCP 서버 - 일정한 IP 주소를 다른 클라이언트에 할당하여 자동으로 설정하게 해준다.
2. DHCP 클라이언트 - 서버에 IP 요청을 하고, IP주소를 부여받으면 다른 호스트와 TCP/IP를 사용해서 통신한다.
    
    → 즉, 할당 받아야 통신이 가능하다는 것

### DHCP 프로토콜의 원리

“임대” 라는 개념

→ IP 주소를 영구적으로 할당하는게 아니라 임대 기간을 명시해서 그 기간동안 IP 주소를 사용하도록 한다.

계속 사용하려면 연장을 해야한다.

그만 사용하려면 반납을 해야한다.
