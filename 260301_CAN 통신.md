## CAN 통신이란?

Controller Area Network

차랸 내 여러 ECU들이 별도의 호스트 컴퓨터 없이 직접 통신할 수 있도록 설계된 통신 프로토콜

ECU란? ELectronic Control Unit

### 왜 자동차 업계에서 CAN을 많이 쓸까?

자동차에 ECU가 탑재되기 시작하면서 서로 데이터를 주고받기 위해 UART을 사용했다

→ UART는 1:1 통신만 가능해 연결선이 매무 많이 필요했다

⇒ 결국 자동차의 무게가 증가하고 제조원가가 상승하게 되었다

### CAN의 특징

여러개의 장치가 하나의 CAN 인터페이스를 통해 통신한다

통신 효율성, 시스템 제어의 안정성, 실시간성, 신뢰성 good

![image.png](attachment:664373dd-7d3b-4ff2-8582-907878da0f52:image.png)

- 여러개의 ECU를 병렬로 연결하여 필요한 장치만 필터링으로 수신
- 여러 노드가 동시에 통신 버스를 공유함 → 중앙제어 없어도 자율적으로 통신 가능
- 두 가닥 신호선만으로 통신 가능

### CAN 통신 프레임 구조

![image.png](attachment:15ad0e0b-2155-4691-ac88-e1d1a1026ce6:image.png)

- SOF :  시작
- ID : 식별자, 우선순위 부여
- Control : 데이터 길이 정보
- Data : 데이터
- CRC : 오류 검출을 위한 체크값
- ACK : 수신 확인용 응답
- EOF : 프레임 종료

### 동작 방법

1. CAN 버스가 다른 제어기에 의해 사용 중인지 확인한다.
2. 비어있을 경우 ID 값을 기준으로 자신에게 필요한 메세지를 식별해 수신한다. (필요하지 않은건 자동으로 무시)
3. 여러 노드가 동시에 메세지를 전송하려 할 경우 ID 값이 낮은 노드가 우선저긍로 버스에 접근한다(오름차순)

### 사용 용도

1. 전기차
2. 의료기기, 로봇 및 드론
3. 차세대 규격 : CAN-FD 확산

> CAN - FD?
> 

Flexible Data RAte

기존 CAN의 한계를 보완한 차세대 확장 규격이다.

최대 64바이트 데이터 전송이 가능하고 8Mbps 까지 가능하다

OTA 업데이트나 배터리 관리 시스템(BMS)에 도입된다
