## 베어메탈이란?

운영체제 없이 MCU에서 직접 실행되는 코드방식

단일루프(while)와 인터럽트를 이용해 동작한다.

하드웨어에 밀접하게 결합되어 있어 낮은 오버헤드와 빠른 실행 속도를 보장한다.

### 장점

- 코드의 크기가 작다
- 메모리 사용량이 많지 않다
- 응답 속도가 빠르다

### 단점

- 태스크 관리 기능이 부족하다
- 유지보수가 어렵다
- 확장성이 부족하다

→ 그래서 단순 임베 시스템이나 센서 데이터 수집에 적합

### RTOS와의 차이점

| 항목 | RTOS | 베어메탈 |
| --- | --- | --- |
| 운영체제 | 있음 | 없음 |
| 멀티 태스킹 | 지원 (태스크 기반) | 미지원 (순차 실행) |
| 우선순위 관리 | 가능 | 불가능 |
| 시간 제어 | 정밀한 타이밍 관리 기능 | 타이머 인터럽트 활용 |
| 복잡도 | 상대적으로 높음 | 낮음 |
| 메모리 관리 | OS가 관리 | 직접 구현해야함 |
| 메모리 사용량 | 상대적으로 많음 | 적음 |
| 개발 난이도 | 비교적 어려움 | 쉬움 |
| 응답 속도 | 보장 가능 | 하드웨어 의존적 |
| 디버깅 | 비교적 복잡 | 단순 |
- RTOS : 여러개의 태스크가 동작해야할 때 스케줄링을 위해 사용한다.

```c
struct gpiod_chip *chip = gpiod_chip_open_by_name(CHIPNAME);
struct gpiod_line *line = gpiod_get_line(chip,GPIO_PIN);

//GPIO를 출력 모드로 설정
if(gpiod_line_request_output(line, "example", 0) < 0){
	perror("Failed to set GPIO line as output");
	gpiod_chip_close(chip);
	return 1;
}

//GPIO 토글
for(int i =0; i<5; i++){
	gpiod_line_set_value(line, 1);
	sleep(1);
	
	gpiod_line_set_value(line, 0);
	sleep(1);
}

gpiod_line_release(line);
gpiod_chip_close(chip);
```

- 베어메탈 : 단순한 어플리케이션이나 리소스가 매우 제한적일 때 사용한다.

```c
void delay(int count){
	while(count--){
		__NOP(); // No Operation (간단한 딜레이)
	}
}

RCC->AHB1ENR |= (1 << 3); // GPIOD 활성화

GPIO->MODER |= (1 << (12 * 2)); //출력 모드 설정

while(1){
	GPIOD->ODR ^= (1 << 12); //출력
	delay(1000000);
}
```

### 두 코드의 차이점

RTOS는 거의 다 함수를 사용한다.

베어메탈은 거의 다 비트마스크를 사용한다. (직접 레지스터 비트 on)
